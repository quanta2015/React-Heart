## 1. 项目概述

### 1.1 项目简介

心理健康测评系统（Heart）是一个面向学校、学生和家长的综合性心理健康评估平台。系统支持学生心理测评、家长心理测评以及教师端数据统计分析功能，旨在帮助学校全面了解学生心理健康状况，及时发现潜在风险并提供干预建议。

### 1.2 用户角色

| 角色           | 说明                | 主要功能                                                   |
| -------------- | ------------------- | ---------------------------------------------------------- |
| 学生 (student) | 参与心理测评的主体  | 完成心理测评问卷、查看个人测评结果和建议                   |
| 家长 (parent)  | 学生监护人          | 完成家长心理测评、查看家庭教育相关评估结果                 |
| 教师 (teacher) | 学校管理者/心理老师 | 查看学生整体统计、风险分布、 individual 学生结果、导出报告 |

### 1.3 核心功能

- **学生测评**：150 题综合心理测评（PHQ9、GAD7、RSES、UCLA、PSS、IAT 量表 + EXT5 扩展题）
- **家长测评**：150 题家庭教育心理测评（六大量表 + 家长专用题库）
- **统计分析**：多维度风险等级统计、领域分析、年级/班级对比
- **报告生成**：PDF 格式测评报告导出

### 3.4 核心算法

#### 3.4.1 风险评分计算

```javascript
// 权重分配
抑郁：0.30
焦虑：0.25
学习压力：0.20
社交：0.15
网络行为：0.10

// 风险等级判定
riskScore < 0.3  → R0 (低风险)
riskScore < 0.5  → R1 (中低风险)
riskScore < 0.7  → R2 (中高风险)
riskScore >= 0.7 → R3 (高风险)

// 特殊规则：PHQ9-09 自杀意念题答案>=3 直接判定为 R3
```

#### 3.4.2 试卷生成算法

**学生试卷**：

- 标准量表题（76 题）：PHQ9、GAD7、RSES、UCLA、PSS、IAT
- EXT5 扩展题（74 题）：按领域和维度随机抽取

**家长试卷**：

- 标准量表题（76 题）：同上
- 家长专用题库（74 题）：从 `type='parent'` 题目中随机抽取

## 7. 安全设计

### 7.1 认证安全

- **JWT Token**：7 天有效期，Bearer Token 方式传递
- **密码存储**：当前为明文存储（生产环境建议改为 bcrypt 加密）
- **角色隔离**：通过 `requireRole` 中间件实现接口级权限控制

### 7.2 数据安全

- **SQL 注入防护**：使用参数化查询
- **XSS 防护**：React 默认转义输出
- **CORS 配置**：允许跨域请求

### B. 家长测评算法说明

#### B.1 领域得分计算

```javascript
// 对每个家长专用领域，计算标准化得分
for each domain in parent_domains:
    domain_score = (sum_score / (item_count * max_per_item)) * 100

// 其中：
// - sum_score: 该领域所有题目的得分总和
// - item_count: 该领域的题目数量
// - max_per_item: 每题最大分值（通常为 4 或 5）
```

#### B.2 家庭环境综合指数

```javascript
// 家庭环境指数 = (家庭沟通 + 亲子关系 + 家庭氛围 + 情感支持) / 4
family_env_index =
  (domain_score["family_communication"] +
    domain_score["parent_child_relationship"] +
    domain_score["family_environment"] +
    domain_score["emotional_support"]) /
  4;
```

#### B.3 家长压力指数

```javascript
// 家长压力 = (家长压力 + 期望压力) / 2
parent_pressure = (domain_score["parent_stress"] + domain_score["parent_expectation"]) / 2;
```

#### B.4 教养方式分类

根据教养方式、管教方式和情感支持三个维度，将家长的教养方式分为五类：

```javascript
function classifyParentingStyle({ parentingStyle, discipline, support }) {
  if (support > 60 && discipline > 60) return "权威型"; // 高支持 + 高要求
  if (discipline > 70 && support < 40) return "专制型"; // 低支持 + 高要求
  if (support > 60 && discipline < 40) return "放任型"; // 高支持 + 低要求
  if (support < 40 && discipline < 40) return "忽视型"; // 低支持 + 低要求
  if (parentingStyle >= 60 && support >= 60) return "权威型";
  if (discipline >= 65) return "专制倾向";
  if (support >= 65) return "放任倾向";
  return "混合型";
}
```

**教养方式类型说明**：

| 类型   | 特征                            | 建议                         |
| ------ | ------------------------------- | ---------------------------- |
| 权威型 | 高支持 + 高要求，民主而坚定     | 继续保持，是最理想的教养方式 |
| 专制型 | 低支持 + 高要求，严格而少温暖   | 增加情感支持，减少强制命令   |
| 放任型 | 高支持 + 低要求，溺爱而少规则   | 建立适当规则，保持一致性     |
| 忽视型 | 低支持 + 低要求，缺乏关注和引导 | 增加陪伴时间，建立基本规则   |
| 混合型 | 各项指标中等，无明显倾向        | 根据具体情况调整教养策略     |

#### B.5 预警机制

系统根据以下规则生成预警信息：

| 预警级别 | 触发条件      | 建议                                       |
| -------- | ------------- | ------------------------------------------ |
| 一级预警 | 家长压力 > 80 | 家长压力过高，建议尽快减压与寻求支持       |
| 二级预警 | 亲子关系 < 40 | 亲子关系偏紧张，建议增加高质量沟通与陪伴   |
| 三级预警 | 家庭氛围 < 30 | 家庭氛围存在冲突风险，建议优先改善沟通模式 |
